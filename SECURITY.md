# Security policy

- Principio del minimo privilegio per ogni tool.
- Nessun segreto nel codice o nei log.
- Azioni irreversibili dietro approvazione esplicita.
- Limiti di costo, iterazioni e tempo applicati dal codice.
- Input esterni trattati come dati non attendibili.
- Prompt injection e tool output injection devono essere testati.
- Gli incidenti devono diventare test di regressione.
